# SeteaseCloudMusic 开发必看

> 仿 Apple Music 风格 Android 音乐播放器 · 后端接口：`nimabi520/api-enhanced`（网易云音乐 API）
>
> 最后更新：2026-04-13

---

## 一、技术栈

| 领域 | 选型 |
|------|------|
| 开发语言 | Kotlin |
| UI 框架 | Jetpack Compose + Material3 |
| 架构模式 | 分层架构 + 按功能分包（Feature-based Clean Architecture） |
| 网络请求 | Retrofit 3.0 + OkHttp + Kotlin Coroutines & Flow |
| 媒体播放 | AndroidX Media3 (ExoPlayer) + MediaSessionService |
| 图片加载 | Coil Compose 2.7 |
| 特效 | Liquid Glass（kyant0/backdrop + shapes） |
| 依赖注入 | 手动 DI（`NetworkModule` 手工组装，未引入 Hilt） |

## 四、开发路线图（接下来的里程碑）

### M3 — 认证系统落地 🔥 当前优先

Auth 模块已有完整的接口定义（`AuthService`、`AuthRepository`）、领域模型和 UseCase，但**缺少核心的运行时胶水层**：

| 编号 | 任务 | 需要新建/修改的文件 | 完成标准 |
|------|------|-------------------|---------|
| 3.1 | **会话持久化** | `core/local/session/CookieStore.kt` · `SessionLocalDataSource.kt` | Cookie（MUSIC_U）可存可取可清 |
| 3.2 | **AuthRepository 实现** | `feature/auth/data/AuthRepositoryImpl.kt` | 组合 AuthService + 本地存储，输出统一登录态 |
| 3.3 | **AuthInterceptor 接入本地会话** | 修改 `core/network/interceptor/AuthInterceptor.kt` | 请求自动从 CookieStore 读取 Cookie 附加 |
| 3.4 | **手机号/邮箱登录 UI** | `feature/auth/presentation/AuthViewModel.kt` · `PhoneLoginScreen.kt` · `EmailLoginScreen.kt` | 可输入→调接口→成功落 Cookie→跳回主页 |
| 3.5 | **验证码登录 UI** | `feature/auth/presentation/CaptchaLoginScreen.kt` | 发送验证码→输入→登录成功 |
| 3.6 | **二维码登录** | `feature/auth/presentation/QrLoginScreen.kt` | key→生成二维码→轮询 801/802/803/800 |
| 3.7 | **游客登录兜底** | 在 AuthRepository 中实现 | 未登录时自动 fallback 游客身份 |
| 3.8 | **登录态刷新** | 修改 `AuthInterceptor` + `RefreshSessionUseCase` | 401/过期自动触发 `refreshLogin`，失败后引导重新登录 |
| 3.9 | **登录入口集成** | 修改 `AppNavigation.kt` | 未登录时搜索/播放前给出登录引导 |

#### 落地顺序建议

```
3.1 会话持久化 → 3.3 拦截器接入 → 3.2 仓库实现 → 3.4 手机/邮箱登录
→ 3.5 验证码登录 → 3.6 二维码登录 → 3.7 游客兜底 → 3.8 登录态刷新
→ 3.9 导航入口集成
```

#### 关键约束（避坑）

- 不要在每次请求前主动重新登录；先读本地会话
- 登录慢是正常现象（加密链路），UI 要有 loading + cancel
- 对密码参数建议 URL 编码或 POST body 传输，避免特殊字符截断
- 二维码轮询必须带时间戳（`timestamp` 参数），防止缓存
- 登录接口避免高频调用，增加最小间隔/退避重试，防止风控

### 鉴权机制

1. 登录成功后后端返回 `Set-Cookie`，其中 `MUSIC_U` 是核心凭证
2. 后续所有需要登录态的请求，在 Header 中附加 `Cookie: MUSIC_U=xxx`
3. `AuthInterceptor` 已实现统一拦截，需接入 `CookieStore` 完成持久化读取
4. 401/登录失效时触发 `refreshLogin`，失败后引导用户重新登录

## 八、编码规范速查

- **分层规则**：`data` → `domain` → `presentation`，依赖方向单向向内
- **Feature 内自治**：每个 feature 包含独立的 data/domain/presentation
- **core 下沉共享**：网络、播放器、核心模型放 `core/`，不混入业务逻辑
- **UseCase 单一职责**：每个 UseCase 只做一件事，通过 `operator fun invoke()` 调用
- **DTO 与领域模型分离**：`data/model/` 放 API 响应 DTO，`domain/model/` 放纯净领域模型
- **状态管理**：ViewModel 持有 `MutableStateFlow<UiState>`，Compose 用 `collectAsState()`
